So how do programmers code?

We have already covered that programming is how users give some input to the computer, but how exactly do they do that? Well, it is through IDEs that the magic happens.

What is an IDE? Put simply, an IDE (Integrated Development Environment) is where programmers write their code and get the final outcome. IDEs allow programmers to consolidate separate pieces of their code while writing their programs. Additionally, IDEs increase producer productivity through their assistance of several features and tools. IDEs range from basic platforms with only a couple of these features to project ready environments with a vast variety of tools.

Some basic features of an IDE include the following:

  1. Text Editor/Syntax-Highlighting
  2. Auto-Completion
  3. Debugger
  4. Compiler
  5. Built-In Plugins and Packages

What is Text Editor:

        Text Editors are pretty much the base of the IDE. This is the feature that actually allows users to play with the code. Text Editors typically have some visual aspects that make coding more comfortable, convenient, and fun! The most basic of these is language-specific syntax highlighting. In other words, if the computer runs across a word or phrase that’s reserved for the specific language (Python in this case), then it will automatically be highlighted in a different color so it stands out.

What is Auto-Completion:

        Well, as the name suggests, this is a feature of IDEs that can pretty much predict what you are writing and give suggestions to complete the code. This feature saves tons of time and reduces chances of having any typos in your code.

What is a Debugger:

        As codes get increasingly complex, it can sometimes be difficult to understand what went wrong if the program doesn’t run. In these cases, debugging tools are there to assist you in identifying and fixing the errors in their code. These tools come in handy more often than you would expect so it is pretty important to get a handle on them.

What is a Compiler:

        The compiler is a key component of an IDE. This is where the programming language that you used to code something is translated into machine language that the computer can understand (ex. Binary code). Fortunately, Python is an interpreted language and not compiled like Java, C, C++. Although compilation does occur when adding the .py extension to your file, there is no formal step like in other languages.

        For more information on how Python works behind the scenes, visit (How does Python work?. A simple explanation of how Python code) There is a good amount of information on the site that the author provides if you are interested.

Built in Plugins/Programs:

For more advanced IDEs, there are numerous additional plugins, or extensions, that further enhance your experience. For this reason, fully understanding what your IDE can offer will likely save a lot of time and frustration.

Variables

So far, you’ve learned how to handle and work with data through the print statement. For example, if your wanted the product of 23 * 12

>>> print("Hello World")
Good Morning

You should get a return value of Good Morning.

Now what if you wanted to run this statement conveniently, everytime you start programming? You could print the statement in this manner but truthfully, that would grow to be quite redundant and tiring.

Thankfully, Python offers an easier way to tackle this problem -- variables. Variables are used to store data values and you can access the values through variables as many times as needed.

Let’s take a look at the previous example once more

>>> greeting = "Hello World"

In this line, we have assigned the string “Hello World” to a variable called greeting. Notice how nothing is being returned, unlike the print statement. This is because we are only declaring a variable for the first time. If, however, we want to get some result, you would simply run the following

>>> print(greeting)
Hello World

Behind the scenes, when you call greeting in the print or at any other point in the program, Python tracks back to the place greeting was first initialized and brings that value down in replacement of the variable.

Also, notice how we don’t wrap greeting in strings. If there were strings around greeting, then Python would treat it like a string and simply return ‘greeting’ rather than the ‘Hello World’ value it actually represents.

Right now, this may seem like more work than to just print “Hello World” and not have to declare any variables, but what if you were further along in your scripting, at a point where you would have to print “Hello World” numerous times? At times like these, variables can show to be quite valuable.

Other instances where we may look to using variables is when we actually don’t want to print anything. Instead, if we only want to initialize the variable that we plan on referring back to at later points in our code, then variables are the way to go.

Strings Introduction

As mentioned before, strings can be quite powerful from its many methods and formatting abilities.

As a refresher, strings are a data type and recognized as a piece of text or characters wrapped in quotes. These can be either single or double quotes.

However, it is recommended that you use double quotes for best practices. This is because if you are not careful, there can be errors produced by using single quotes. For example:

phraseA = "What's Up"
praseB = 'What's Up'

Looking at the above code, hopefully you will notice that some of the string in phraseB is not highlighted the way the beginning is. This is because, when you use single quotes, an apostrophe (the quote after ‘What’ in phraseB) acts as a closing quote. However, we didn’t mean for that and still had more text that is encased by another quote at the end. All of this confuses the computer and you end up with an error

To combat this, using double quotes is the best way to go. But if you still want to use single quotes, you can implement an escape sequence

phraseC = 'What\'s Up'

As you can see, this works, just isn’t as nice and convenient as simply using double quotes.

Strings are immutable

Before we get started on some methods, it is important to understand that strings are immutable data types. This means that once a string is initialized and assigned to a variable, it cannot be changed. Thus, you cannot overwrite any part of that string. However, you can assign the variable again to a different string.

>>> name = "John"
>>> print(name)
John
>>> name = "Sara"
>>> print(name)
Sara

Although it may look like you are changing the value of ‘John’ to ‘Sara’, you are actually creating a new variable, only with the same name.

You can check this out in further detail using the id() function which should return each of the variables’ memory addresses

>>> name = "John"
>>> print(id(name))
4311978032
>>> name = "Sara"
>>> print(id(name))
4311978224

Since strings are immutable, you can see that the addresses are both different.

A clearer way to see that Strings are immutable data types is through indexing which you will see in the Indexing tutorial.

String Indexing

Strings consist of a grouping of characters; they can be letters, numbers, and even special characters. Each character in a string also has its own position/index. Think of it this way: all of the characters in the string are standing in line, the first one has the first position, while the last one has the last position number. However, one key difference to note her is that the first character index is not 1 but instead 0.

>>> word = "Python"

The first letter in this example is ‘P’ and it’s index is 0.

The second letter ‘y’ has an index of 1.

This pattern continues to the end of the string (last letter ‘n’ is at 5th index).

In Python, we can access certain positions of our string using indexes.

>>> print(word[0])
P
>>> phrase = "This is a string with spaces!"
>>> print(phrase[5])
i

You may have noticed that in this example, the space was also assigned a position.

So far so good! But what if we had a very long string and were told to get the very last character. Fortunately, Python gives us a way to more conveniently access either side of a string

>>> print(phrase[-1])
!

Using the negative sign before the number when indexing tells Python to start counting from the left last character. In our case, the last character is an exclamation point ‘!’.

Now that you have an idea of how to retrieve a single character, can you think of how to get a certain group of characters?

Again, indexing helps us with that…

>>> thought = "I love to code!"
>>> print(thought[2:6])
love
>>> print(thought[-5:-1])
code

The basic syntax for this is as follows: string[start:end + 1])

You may have noticed in the above example that when using this method, Python includes the character at the start index and ends at the character before what is at the end index.

Another option is to actually ask for the starting index of a substring using the index() function.

>>> print(thought.index("code"))
10

IF, ELSE and ELIF Statements

In the previous tutorials, we have looked at the different conditional statements and the True or False aspect of it all. Now, let’s put that knowledge to use and actually do something based on that boolean value.

The IF and ELSE blocks are used to accomplish this:

x = 10
if x < 50:
   print("Your number is less than 50")
else:
   print("Your number is greater than 50")
print("Done")

If you run this, you will see that the return answer is “Your number is less than 50” followed by “Done”. Looking at the if statement, we see a conditional ‘x < 50’. Here, Python traces back to when x was declared and remembers that it is equal to 10. Now computing that x (10) is in fact less than 50, the statement is True. Python then runs whatever is inside of the if block (which in this case is our print statement) and then skips the else statement and goes down to print “Done”.

It is important to recognize that only one of the statements will run (either the if or the else). If the condition is true, the body of the if block will run and when the condition is false, the else block will run.

Now that you hopefully understand how if and else blocks work, let’s go back to our example. What if we change the value of x to 50 and run the code:

x = 50
if x < 50:
   print("Your number is less than 50")
else:
   print("Your number is greater than 50")
print("Done")

Return:

Your number is greater than 50

Done

There’s a bug in our code! x is equal 50 not greater. Fortunately, in Python, there’s another statement we can add in our code to make this work out:

x = 50
if x < 50:
   print("Your number is less than 50")
elif x == 50:
   print("Your number is equal to 50")
else:
   print("Your number is greater than 50")
print("Done")

Return:

Your number is equal to 50

Done

Keep in mind that only one of these statements will run which one that may be is based on the condition in the if block and later in the elif statements. Also, there can only be one if statement that starts it off. You can, however, have as many elif statements as you feel necessary but end with only one else statement.

Think of these statements like a sandwich. You only put one bread on top (that serves as the if), as many layers in the middle, which help add more to your sandwich (elif statements), and finally, one slice of bread on the bottom to finish it off (else).

Also, you may have noticed that while the if and elif statements have conditions, there is nothing for the else statement. This is because the else block serves as a sort of “if all else fails then run what’s in the else block”.

Lists

What are lists, how are they created, and what are their functions?

Recall the four main data types: strings, integers, floats, and booleans. Similar to data types, there are also what we call data structures. Lists is one of the most frequently used and versatile of the four data structures (lists, tuples, sets, dictionaries).

Creating a List:

As the name says, a list is composed of items/elements. In python, we wrap our list in square brackets ‘[]’ and separate each item with a comma. There is no limit on the number of items in a list and they can include any of the data types.

#empty list
my_list = []

#list of strings
my_list = [123, 234, 654, 760]

#list of integers
my_list = ["sun", "moon", "clouds"]

#list of mixed data-types
my_list = [1, "hi", 3.14, True]

We can also include lists inside of the list. These are called nested lists.

#nested list
my_list = ["happy", ["a", "b", "c"], [1, 2, 3]]

List Indexing

Similar to string indexing, we can use indexing to access a given element. Remember, the first indices is 0.

my_list = ["Python", "rocks", [2, 4, 6, 8]]
print(my_list[0])

Append/Extend - Adding to the end of a list

There are two ways to add to the end of a list: append() and extend(). While append only allows you to add one item at a time, extend lets you add as many items as you want to the list.

[0, 2, 4, 6, 8, [10, 12]]

As you can see, when you use append() and try to add more than one item, it is treated as a nested list rather than separate elements.

Let’s use exten() and see what we get:

even = [0, 2, 4, 6]

[0, 2, 4, 6, 8, 10, 12]

List Comprehensions: A crisp and dynamic way to create lists

List comprehension is simply a way to create a new list using for loops and possibly if statements.

Format: listname = [<expression>]

Let’s start with for loops:

pows = [x**2 for x in range(1, 11)]
print(pows)

[1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

From this example, you can probably see that this example is a concise way to write something like this:

pows = []
for x in range(1, 11):
        pows.append(x**2)
print(pows)

[1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

You can make the list more precise to your needs with an additional if statement.

pows = [x**2 for x in range(11) if x > 4]
print(pows)

[25, 36, 49, 64, 81, 100]

Loops

Loops are used to iterate through a block of code based on a given expression

While Loops

In while loops, the code in the loop runs if the condition is true.

Let’s take a look at an example:

n = 0
while n <= 6:
 print(n)
 n += 1

In this example, note that  n += 1  is just a more concise way of writing n = n + 1

Format:

while <expression>:

        <code to run if expression is true>

NOTE: make sure there is a mechanism to end to loop. Otherwise, you could end up with an infinite loop (it won’t stop running on your console) For example, if instead of using n+=1, you say n-=1, you will be stuck in an infinite loop since n will always be less than 6.

Break & Continue

Sometimes, you want to exit the loop, even if the condition still holds true. To achieve this, we use the break command.

n = 0
while n <= 6:
        print(n)
        if == 3:
                break
        n += 1

Other times, you may want to skip an iteration and continue on with the loop. You can use the continue command for this.

n = 0
while n <= 6:
        print(n)
        if == 3:
                continue
        n += 1

Feel free to play around with these two commands and see what you get!

Similar to the way an if statement works, when the while condition is false, you can use an else statement to run something else.

n = 0
while n < 6:
        print(n)
        n += 1
else:
     print("n is greater than 6")

For Loops

For loops are slightly different. They iterate through a sequence (this can be a string, list, tuple, dictionary, etc). The program will end once the computer finishes iterating through the sequence or there is a break in the loop.

If you are not comfortable with data structures yet, I recommend you take a look at the lists, tuples, and dictionaries tutorials.

names = ["sally", "andy", "bob"]
for name in names:
        print(name.capitalize())

Format:

for <iterator> in <iterable>:

        <code to run while sequencing through iterable>

In a for loop, the iterator is just a placeholder while you run the loop to go through the iterable (whatever string, list, tuple, etc. that you want to go through). As you can see in the example above, you do not need to initialize an iterator beforehand.

Here is another example:

even = []
for i in range(10):
        if i % 2 == 0:
                even.append(i)

You can also add if statements or even the break and continue commands we looked at above to add more context to your for loops.

Functions

What are functions? What purpose do they serve in programming? And, how do we use them? These are all common questions that you should be comfortable in answering by the end of this tutorial. If you have read the String Functions tutorial or have some basic knowledge in Python, then you have seen a few methods (another word for functions) already.

Functions are blocks of code that you have already written out and can refer back to later on. They are similar to variables in the sense that you can save them using a specific name and later when you refer to it, the computer will go up to where it was first declared.

Let’s create a function:

def hello(name):
     greeting = "Hello {}".format(name.capitalize)
        return greeting

print(hello("bob"))
print(hello("sally"))
print(hello("tommy"))

Format:

def <function_name>(<argument>):

        <what you want function to do>

        return <what you want function to return>

In a function, the function name serves as a ‘variable name’ you can use when calling it later on. As you can see in the example, you can run the function as many times as you want, each time passing a different argument (the argument is what we called name in the function).

Also, note that name is only used in the function and you do not declare it outside of the function.

You can also pass any data structure/type as you’d like for your argument.

def subjects(classes):
        for class in classes:
                return class
my_classes = ["math", "english", "science", "history"]


print(subjects(my_classes))

OOP

Python is an object oriented programming (oop) language. In other words it is all about the use of objects and classes to approach different problems. OOP is vital to creating concise and powerful code.

Objects are simply a collection of data. The program as a whole orchestrates these many objects to get what you need. We have been using objects all along: strings, integers, dictionaries, lists, etc.

Some other key words to think about are:

  • Class -- the shape of an object/template
  • Method -- a defined capability of the class
  • field/attribute -- a bit of data in the class
  • Object -- an instance of the class (instance and objects can be used interchangeably)

You can think of Strings as a whole as the class and each string you create is an object (str1, str2, str3 are all different objects). Methods, as we have said before, are str.upper(), str.join(), etc.

Ex.

  • 3 -- instance of the integer class
  • “python” -- instance of the string class
  • All of the following are then methods of the string class:
  • capitalize()
  • lower()
  • islower()
  • upper()
  • isupper()
  • endswith()
  • 10.45 -- instance of the float class
  • [1, 2, 3, 4] -- instance of the list class

Up until now, we have used predefined classes and objects, but now, we will create our own class.

When we look at the format of a class, it is pretty similar to that of a function:

class PartyAnimal:  #PartyAnimal is the template for following objects
        x = 0    #Each PartyAnimal object will have its own bit of code
        def party(self): #This is code each PartyAnimal obj. will have
                self.x = self.x + 1
                print("So far {}", self.x)
anim = PartyAnimal()   #Construct the object and store it in a variable
                  #anim is object created through PartyAnimal template
anim.party()     #tells object to run party() code within PartyAnimal
anim.party()     #the parameter that replaces 'self' is anim
