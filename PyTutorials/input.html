User Input

In Python, in order to add more interaction to your experience, we have a built in function called input. Let’s take a look at an example:

name = input("Enter your name: ")
print(name)

Try running this on your console and you will see that in your console, the computer will ask you to “Enter your name: ” The computer will then save the value you have entered into the variable name. Following that, the computer will print your name. Pretty cool, right?

An aspect of the input function that you should recognize is that the value you enter for name can change. Unlike when you actually declare a variable, when you use the input(), each time you run the program, your variables will reset. In other words, when you hit run on your IDE and you are asked to enter your name, the name you enter will be used throughout the code, or at least whenever mentioned. However, if you try to run your program again, the computer will no longer have a value stored in name and will ask you to enter it again.

What if you wanted to enter a number rather than a String? Let’s try it out:

age = input("Enter your age: ")
print(age)

Enter your age: 100

100

Try decreasing the age by two years and see what you get:

age = input("Enter your age: ")

age = age - 2
print(age)

Enter your age: 100

 

Traceback (most recent call last):

  File "random.py", line 2, in <module>

    age = age - 2

TypeError: unsupported operand type(s) for -: 'str' and 'int'

It’s an error! It reads “unsupported operand type(s) for -: 'str' and 'int'” 

We can debug this by taking a look at taking a look at the type() of the variable age:

print(type(age))


<class 'str'>

So even if you entered an integer, the computer still thinks that age is a string. In Python, we can use the int() built in function to convert the string to an integer.

age = int(input("Enter your age: "))
age = age - 2
print(age)

Enter your age: 100
98
