Strings Introduction

As mentioned before, strings can be quite powerful from its many methods and formatting abilities.

As a refresher, strings are a data type and recognized as a piece of text or characters wrapped in quotes. These can be either single or double quotes.

However, it is recommended that you use double quotes for best practices. This is because if you are not careful, there can be errors produced by using single quotes. For example:

phraseA = "What's Up"
praseB = 'What's Up'

Looking at the above code, hopefully you will notice that some of the string in phraseB is not highlighted the way the beginning is. This is because, when you use single quotes, an apostrophe (the quote after ‘What’ in phraseB) acts as a closing quote. However, we didn’t mean for that and still had more text that is encased by another quote at the end. All of this confuses the computer and you end up with an error.

To combat this, using double quotes is the best way to go. But if you still want to use single quotes, you can implement an escape sequence:

phraseC = 'What\'s Up'

As you can see, this works, just isn’t as nice and convenient as simply using double quotes.

Strings are immutable

Before we get started on some methods, it is important to understand that strings are immutable data types. This means that once a string is initialized and assigned to a variable, it cannot be changed. Thus, you cannot overwrite any part of that string. However, you can assign the variable again to a different string.

>>> name = "John"
>>> print(name)
John
>>> name = "Sara"
>>> print(name)
Sara

Although it may look like you are changing the value of ‘John’ to ‘Sara’, you are actually creating a new variable, only with the same name.

You can check this out in further detail using the id() function which should return each of the variables’ memory addresses:

>>> name = "John"
>>> print(id(name))
4311978032
>>> name = "Sara"
>>> print(id(name))
4311978224

Since strings are immutable, you can see that the addresses are both different.

A clearer way to see that Strings are immutable data types is through indexing which you will see in the Indexing tutorial.
