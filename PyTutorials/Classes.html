Classes

Python is an object oriented programming (oop) language. In other words it is all about the use of objects and classes to approach different problems. OOP is vital to creating concise and powerful code.

Objects are simply a collection of data. The program as a whole orchestrates these many objects to get what you need. We have been using objects all along: strings, integers, dictionaries, lists, etc.

Some other key words to think about are:

  • Class -- the shape of an object/template
  • Method -- a defined capability of the class
  • field/attribute -- a bit of data in the class
  • Object -- an instance of the class (instance and objects can be used interchangeably)

You can think of Strings as a whole as the class and each string you create is an object (str1, str2, str3 are all different objects). Methods, as we have said before, are str.upper(), str.join(), etc.

Ex.

3 -- instance of the integer class

“python” -- instance of the string class

  • All of the following are then methods of the string class:
  • capitalize()
  • lower()
  • islower()
  • upper()
  • isupper()
  • endswith()

10.45 -- instance of the float class

[1, 2, 3, 4] -- instance of the list class


Up until now, we have used predefined classes and objects, but now, we will create our own class.

When we look at the format of a class, it is pretty similar to that of a function:

class PartyAnimal:  #PartyAnimal is the template for following objects
        x = 0    #Each PartyAnimal object will have its own bit of code
        def party(self): #This is code each PartyAnimal obj. will have
                self.x = self.x + 1
                print("So far {}", self.x)
anim = PartyAnimal()   #Construct the object and store it in a variable
                  #anim is object created through PartyAnimal template
anim.party()     #tells object to run party() code within PartyAnimal
anim.party()     #the parameter that replaces 'self' is anim

Try running this and see what you get!

Similar to finding types of different string instances, we can find the type of anim. We can also use dir() to see the different methods in of anim:

print(type(anim))
print(dir(anim))

<class '__main__.PartyAnimal'>
['__class__', ... ,'party', 'x']
